# מיפוי שדות ייבוא - CRM לכספית

## מבנה הקובץ מה-CRM

| עמודה ב-CRM | עמודה בכספית | הערות |
|-------------|--------------|-------|
| מספר המסמך | invoice_number | |
| סוג מסמך | document_type | דורש תרגום (ראה למטה) |
| שם הלקוח | customer_name | להתאמה מול רשימת לקוחות |
| תאריך המסמך | date | פורמט DD.MM.YYYY → YYYY-MM-DD |
| סטטוס | document_status + payment_status | דורש תרגום |
| חשבונית ללא מע"מ | amount_before_vat | |
| מוכר מע"מ | vat_amount | |
| חשבונית רגילה | amount | סכום כולל מע"מ |

## תרגום סוגי מסמכים

| CRM | כספית |
|-----|--------|
| חשבון עיסקה | invoice |
| חשבונית מס | tax_invoice |
| חשבונית מס קבלה | tax_invoice_receipt |
| קבלה | receipt |
| חשבונית זיכוי | credit_note |

## תרגום סטטוסים

| CRM | document_status | payment_status |
|-----|-----------------|----------------|
| מסמך סגור | closed | paid |
| פתוח | open | pending |

## לוגיקת קישור אוטומטי

קישור יתבצע **רק** אם:
1. יש התאמה יחידה של לקוח + סכום
2. תאריך מסמך הסגירה (חשבונית מס/קבלה) אחרי תאריך חשבון העיסקה

### זרימות נתמכות:
```
חשבון עיסקה → חשבונית מס קבלה (סוגרת)
חשבון עיסקה → חשבונית מס → קבלה (סוגרת)
```
